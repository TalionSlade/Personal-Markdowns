Figure 06 — Component 4

Counterfactual Reflection Module

Adaptive threshold & synthetic-row injection that keeps the SCM honest under shift

Causa · §E1

Trigger algorithm

1
Compute expected outcome y_hat under do(X = chosen_action)
Plug-in from the back-door estimator using the current observation history.
2
Compute observed outcome y_obs from the tool output
Recorded in the step's observation row before reflection runs.
3
Compute discrepancy δ = |y_hat − y_obs|
The size of the surprise — how badly the SCM mis-predicted this step.
4
Compare δ to threshold θ
Default AdaptiveMedianThreshold: θ = median(|δ_history|) + ε.
δ > θ — TRIGGER
Call LLM with [CAUSA::counterfactual] prompt. Parse alternative-action estimates. Append synthetic SCM-shaped rows to the history under reflection.update.triggered = True. FIRE
δ ≤ θ — SKIP
Record reflection="no update" in the trace step. The SCM was within tolerance — no synthetic injection. QUIET
Threshold
θ ← median(|δ_1|, …, |δ_n|) + ε ε = 1e-3
Synthetic row format
row = { …state, X = a, Y = y_hata, _synthetic = True }

Why this matters under shift

Robustness argument
Under distribution shift, the empirically-estimated effect P̂(Y|X,Z) drifts away from the test distribution P(Y|X,Z). The CRM detects the drift through δ, then patches the history with counterfactual estimates from the LLM — a hybrid that anchors decisions to causal structure while admitting that the data may be stale.
