Figure 01 — Architecture

Causa: Hexagonal System Overview

Pearl-style causal planning for LLM debugging agents — Ports & Adapters layering

Causa · Dissertation
⬢
Core Domain
src/causa/core/

Pure, dependency-free representation of Pearl's structural-causal-model algebra. Knows nothing about LLMs, files, or DoWhy — every adapter depends on the core, never the reverse.

graph.py — Acyclic DAG with invariant checks (no self-loops, no cycles)
scm.py — SCM + SCMBuilder: variables × equations × graph
identifiability.py — Back-door criterion identification (minimal sets)
intervention.py — Graph mutilation under do(X = x)
variables.py — Typed domains: BINARY, CATEGORICAL, CONTINUOUS
⊕
Application Layer
src/causa/{agents,planning,reflection,extraction,evaluation,telemetry}/

Use-case orchestration. Agents drive the decision loop; planning & reflection wire causal mechanics into action selection; evaluation runs the distribution-shift protocol; telemetry persists auditable traces.

agents/ — CausalPlanningAgent, ReActAgent, CoTAgent, NoMemoryAgent
planning/ — DoWhyActionScorer, LLMActionScorer cold-start fallback
reflection/ — Counterfactual reflection module & thresholds
extraction/ — LLMGraphExtractor for novel-domain DAGs
evaluation/ — EvaluationRunner, partitioner, BCa & permutation stats
telemetry/ — JSON Lines trace writer + post-hoc TraceAudit
⇆
Ports & Adapters
src/causa/{ports,adapters}/

Ports declare narrow contracts (Protocols); adapters supply concrete implementations. Every dissertation ablation is a single-line adapter swap in causa.cli.runtime.build_agent — no plumbing rewrites.

ports/llm.py ⇆ adapters: Anthropic, MockLLM
ports/scorer.py ⇆ adapters: DoWhy, LLM-only, hybrid
ports/estimator.py ⇆ adapters: linear regression, DML
ports/history.py ⇆ adapters: pandas, in-memory
ports/tool.py ⇆ adapters: stubbed debugging tools (8 actions)
ports/threshold.py ⇆ adapters: static, adaptive-median
Driving / Driven Boundaries (CLI ↔ Domain ↔ External Services)
cli/main.py
