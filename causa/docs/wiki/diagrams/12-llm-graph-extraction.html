Figure 12 — Component 1

LLM Causal-Graph Extractor

Cold-start path for novel domains — used in §N3 domain-transfer experiments

Causa · §C1
1
Input

A variable list + a free-form domain description (Markdown or prose).

variables = [
  "incident_severity",
  "team_response_time",
  "mitigation_action",
  "downtime_minutes",
]
description = """
Site reliability postmortems.
Severity drives both response speed
and the chosen mitigation. The
mitigation determines downtime …
"""
2
LLM proposal

Render the [CAUSA::extract_edges] prompt & ask for a sparse DAG. JSON-only response, no prose.

{ "edges": [
  {"source": "incident_severity",
   "target": "team_response_time",
   "semantics": "severe → faster"},
  {"source": "incident_severity",
   "target": "mitigation_action"},
  {"source": "mitigation_action",
   "target": "downtime_minutes"},
  {"source": "downtime_minutes",
   "target": "incident_severity",
   "semantics": "feedback?"}
]}
3
Cycle & identity filtering

Build a CausalGraph incrementally. Drop edges that introduce cycles, reference unknown variables, or form self-loops.

for edge in proposed:
  if edge.target in unknown_vars:
    rejected.append(edge); continue
  try:
    graph.add_edge(CausalEdge(...))
  except GraphInvariantError:
    rejected.append(edge)
  else:
    accepted.append(edge)
4
Output

A validated DAG + the accepted/rejected ledger. The ledger is what the LLM-extraction-quality §N3 eval grades.

graph, accepted, rejected = (
  extractor.extract(
    variables=variables,
    domain_description=description,
  )
)

# graph: validated CausalGraph
# accepted: 3 edges
# rejected: 1 (cycle)
