Figure 03 — Algorithm

The Do-Calculus Action-Scoring Pipeline

Identify → Estimate → Score — the causal core of DoWhyActionScorer

Causa · §D1
SCM + candidate action a ⟶ ActionScore (point estimate + rationale)
1
Identify
causa.core.identifiability

Apply Pearl's back-door criterion to the SCM graph for the (action, outcome) pair. Returns the minimal admissible set Z — or signals non-identifiability.

result = identify_effect( graph, treatment="X", outcome="Y", ) result.identifiable # bool result.adjustment_set # frozenset[str] result.method # "back-door"
2
Estimate
causa.adapters.estimators.linear_regression

Plug Z into the back-door adjustment formula. Adapter implements EffectEstimator; swappable for DML or doubly-robust estimators without touching scorer.

est = LinearRegressionEstimator() est.estimate( data, treatment="X", outcome="Y", treatment_value=a, adjustment_set=Z, ) # → EffectEstimate(value, ci, method)
3
Rank
causa.planning.dowhy_scorer

For every action candidate, take E[Y | do(X=a)] − E[Y | do(X=baseline)]. Convert to ActionScore with the back-door set surfaced in rationale.

for c in candidates: yhat = estimator.estimate(...) yield ActionScore( action=c, score=yhat.value, rationale=f"via back-door {sorted(Z)}", adjustment_set=Z, )
4
Audit
causa.telemetry.events / TraceAudit

Every score lands in the JSON Lines trace with scores, chosen_action, adjustment_set, rationale. Post-hoc auditor classifies steps as cold-start vs steady-state.

audit = audit_trace(trace_path) audit.steady_state_steps # via back-door audit.cold_start_steps # [cold-start LLM] audit.reflection_triggers # synthetic rows
