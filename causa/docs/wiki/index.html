The single biggest design idea

The agent's outer loop — step budget, state transition, trace emission — is shared across all six ablation arms in BaseAgent. Only the _choose_action, _after_step, and _next_state hooks vary. This is the seam the dissertation's §J3 ablations cut along, and it's what makes the comparison strictly ceteris-paribus. 

for step_index in range(self._step_budget):
    ctx    = self._snapshot_context(step_index, state)
    choice = self._choose_action(candidates, ctx, task)   # 🔧 varies
    output = self._tools[choice.action.name](...)
    self._after_step(step=step, task=task, state=state)   # 🔧 varies
    state  = self._next_state(prev_state=state,           # 🔧 varies
                              observation=obs)

The single most important file

causa.planning.dowhy_scorer.DoWhyActionScorer — it is the operational core of Component 2. It identifies the back-door set, fits the plug-in estimator, and emits a typed ActionScore with the adjustment set surfaced in its rationale for downstream auditing. 

def score(self, candidates, *, state, history):
    Z = identify_effect(self.scm.graph,
                        X=self.action_var,
                        Y=self.outcome_var).adjustment_set
    out = []
    for c in candidates:
        est = self.estimator.estimate(
            data=history, treatment=self.action_var,
            outcome=self.outcome_var,
            treatment_value=c.name,
            adjustment_set=Z,
        )
        out.append(ActionScore(
            action=c, score=est.value,
            rationale=f"via back-door {sorted(Z)}",
            adjustment_set=Z))
    return out
